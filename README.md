# 🎹 OtoCopy — J-POP耳コピ・ピアノ練習アプリ (MVP)

YouTubeのJ-POPリンクを貼ると、その曲のコード進行を自動推定し、**原曲の再生位置に同期して「今鳴っているコード・ベース音・ピアノの押さえ方・コード音」を同時に確認できる**耳コピ補助アプリです。

メロディーは耳コピできるけれど、ベース音やコード感が分からず「ピアノで弾くと原曲っぽくならない」人のためのツールです。完全な自動採譜は目指していません。**「リンクを貼ったら、とりあえず叩き台として使える70点のコード進行が出てくる」**体験を最優先にしています。

## 起動方法

```bash
npm install
npm run dev        # 開発サーバー (http://localhost:3000)

# 本番ビルド
npm run build
npm start

# コアロジックのユニットテスト
npm test
```

Node.js 20以上を推奨。APIキーなどの環境変数は不要です。

> **注意**: コード推定はサーバー側でYouTube (oEmbed) と外部コード譜サイトにアクセスします。外部ネットワークに出られない環境では、自動的にJ-POP頻出進行のフォールバックが使われます。

## 使い方

1. YouTubeのURLを貼って「解析する」を押す
2. 数十秒でコード候補がタイムライン化されて表示される
3. 再生モードを選んで練習する
   - **原曲のみ**: YouTubeを再生。コード表示・鍵盤表示が再生位置に同期
   - **原曲＋コード**: YouTubeに合わせてアプリ側でもコード音（簡易ピアノ音）を同期再生。コードが合っているか耳で確認できる
   - **コードのみ**: YouTubeを止めてコード進行だけをピアノ音で再生
4. 違うところがあれば直す（すべて自動でローカル保存されます）

### 編集操作

| 操作 | 方法 |
|---|---|
| コードの選択・編集 | タイムラインのブロックをクリック → 編集パネルでコード名/ルート/種別/ベース音/開始/終了/メモを変更 |
| コード切り替わり位置を打つ | 再生しながら `N` キー（または「✂ ここで次のコード」） |
| 現在位置にコード追加 | `A` キー（または「＋ 現在位置にコード追加」） |
| タイミングの微調整 | タイムラインのブロック左端をドラッグ |
| 選択コードの開始/終了を現在位置に | `S` / `E` キー |
| ループ区間 | `[` `]` で開始/終了を現在位置に、`L` でON/OFF。秒数の直接入力も可 |
| 再生/停止・シーク | `Space` / `←` `→` (2秒) |

ユーザーが編集したコードは「手動確定」として最優先で扱われ、緑のドットでマークされます。

## MVPでできること

- YouTubeリンク入力・埋め込み再生・タイトル/チャンネル名/動画長の取得
- 動画タイトルからの曲名・アーティスト名推定（MV / Official Video / 歌ってみた 等の不要語除去）
- 外部コード譜の検索・抽出・複数ソース照合による初期コード進行の自動生成
- 外部情報が見つからない場合のフォールバック（王道進行・カノン進行・小室進行の仮置き）
- 動画長に合わせたコードタイムライン自動生成
- YouTube再生位置と同期したコード表示・ピアノ鍵盤表示（左手=青 / 右手=緑、ベース音強調）
- オンコード（分数コード）対応: `G/B` なら左手ベースは `B`
- コードごとの簡単な日本語解説と出典バッジ（Nソース一致 / 外部コード譜 / 仮置き / 手動確定）
- Web Audioによるコード音再生（両手 / 左手のみ / 右手のみ、音量・音の長さ調整）
- 原曲＋コード同期再生（シーク・停止・ループ時に古い音が残らない設計）
- コードのみ再生（AudioContextクロックによる独立トランスポート）
- 任意区間ループ（全モード対応）
- コード・タイミングの編集、キーボードショートカット
- 曲ごとのプロジェクト単位でのローカル保存（localStorage、自動保存）

## コード推定の仕組み

```
YouTube URL
  → videoId抽出
  → oEmbed / watchページからタイトル・チャンネル名・動画長を取得
  → タイトルから曲名・アーティスト名を推定（不要語除去 + J-POPでよくある表記パターン）
  → 検索クエリ生成（「{曲名} {アーティスト} コード」など複数）
  → 外部コード譜を並列検索
      - ChordWiki サイト内検索
      - Web検索(DuckDuckGo)経由で既知のコード譜サイト（J-Total Music / 楽器.me / U-FRET等）
  → 各ページからコードを抽出（3段構えの汎用エクストラクタ:
      ①class="chord"等のマークアップ ②[C]ブラケット表記 ③コードが密集した行）
  → 複数ソースを照合（コード語彙のJaccard類似度 + バイグラム一致）
      - 最も信頼できるソースの進行を採用
      - 複数ソースで一致するコードは信頼度を上げて「Nソース一致」表示
  → 見つからなければJ-POP頻出進行（王道/カノン/小室）を低信頼度で仮置き
  → 動画長に合わせてタイムライン化
      - 1コード≒1小節（約2.2秒）を目安に進行の繰り返し回数を決めて等間隔配置
  → ユーザーが再生しながらタイミング・コードを修正（手動確定が最優先）
```

信頼度は小さなバッジとして参考程度に表示し、大きく主張しません。

## 現在の制限

- **タイミングは目安**: 外部コード譜には秒情報がないため、初期配置は等間隔です。イントロ・間奏のズレはタップ修正（`N`キー等）で直す前提です
- **音声解析は未実装**: 現在は外部コード譜とテキスト情報のみ。原曲の音からのキー/コード検出は行っていません
- **キー違い・カポ表記**: 移調されたソースは照合スコアが下がって選ばれにくくなる仕組みですが、キーを揃えるアラインメントはまだありません
- **外部サイトの構造変化に弱い**: スクレイピングベースのため、サイト改修で取得できなくなることがあります（その場合はフォールバック進行になります）
- **コード音は簡易シンセ**: オシレータベースのピアノ風音色です（差し替えやすいよう `ChordPlayer` に分離済み）
- **保存はブラウザローカルのみ**: localStorage保存のため、別端末とは共有できません
- **著作権への配慮**: 表示されるコード候補は外部の有志コード譜等に基づく参考情報です。出典リンクを画面に表示しています

## 技術構成

- **Next.js 15 (App Router) + TypeScript + React 19**、外部UIライブラリなし
- コード推定（`src/lib/chordSources/`）・コード理論（`src/lib/chords.ts`）・タイムライン（`src/lib/timeline.ts`）・音声（`src/lib/audio.ts`）・同期エンジン（`src/hooks/useSyncEngine.ts`）・保存（`src/lib/storage.ts`）を分離
- 同期は「モードごとに1つのクロック（YouTube再生位置 or AudioContext時刻）」に統一し、rAFループでコード境界・ループ・シークを検知。シーク/ループ/停止時は必ず発音を止めてから再トリガー
- `npm test` でボイシング・パース・タイトル推定・抽出・照合・タイムラインの41ケースを検証
