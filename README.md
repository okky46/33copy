# 🎹 OtoCopy — J-POP耳コピ・ピアノ練習アプリ (MVP)

YouTubeのJ-POPリンクを貼ると、その曲のコード進行を推定し、**原曲の再生位置に同期して「今鳴っているコード・ベース音・ピアノの押さえ方・コード音」を同時に確認できる**耳コピ補助アプリです。

メロディーは耳コピできるけれど、ベース音やコード感が分からず「ピアノで弾くと原曲っぽくならない」人のためのツールです。完全な自動採譜は目指していません。**実際の曲に基づいた70点の叩き台を出し、ユーザーがすぐ確認・修正できる**ことを最優先にしています。

## プロダクト方針

- **根拠のないコード進行は表示しない**。外部コード譜・音源解析・手動入力・保存済みデータのいずれかの根拠がある場合のみコード候補を出します
- 曲と無関係な定番進行（カノン進行など）の仮置きは**しません**。取得できなかった場合は正直に「十分なコード候補を取得できませんでした」と表示し、音源解析・曲名修正・手動入力へ誘導します
- 外部コード譜は正解扱いせず（キー違い・カポ・簡略コードがある前提）、複数ソースの一致と音源解析との照合で信頼度を決めます
- タイミングは動画長への均等割りではなく、**BPM・拍・小節グリッド**（音源解析の実測、なければ低信頼と明示した仮定グリッド）に沿って配置します
- ユーザー編集が常に最優先です

## 使い方

1. YouTubeのURLを貼って「解析する」を押す
2. 外部コード譜が見つかれば、コード候補がグリッドに沿ってタイムライン表示される
3. **「🎵 音源を解析」から音源ファイル (mp3 / wav / m4a) をアップロード**すると、BPM・拍・小節グリッド・コード候補を曲の音から推定し、タイミングが曲に合う
   - 外部コード譜があるとき: 実測グリッドに再配置し、音源と矛盾するコードに「⚠ 要確認」を付ける
   - 外部コード譜がないとき: 音源解析のコード候補をそのまま叩き台にする
4. 再生モードを選んで練習する
   - **原曲のみ** / **原曲＋コード**（同期してコード音を鳴らす）/ **コードのみ**
5. 違うところを直す（すべて自動でローカル保存）

### 編集操作

| 操作 | 方法 |
|---|---|
| コードの選択・編集 | ブロックをクリック → コード名/ルート/種別/ベース音/開始/終了/メモ |
| コード切り替わり位置を打つ | 再生しながら `N`（「✂ ここで次のコード」） |
| 現在位置にコード追加 | `A` |
| タイミングの微調整 | ブロック左端をドラッグ（拍/小節スナップ対応） |
| スナップ | OFF / 拍 / 小節 を切り替え |
| 全体シフト | ±0.1s / ±0.5s ボタン、`Shift+←→` |
| BPM手動変更 | BPM入力でグリッドを引き直し（グリッドがない場合は新規作成） |
| 小節頭=現在 | 最初の小節頭を現在の再生位置に合わせてグリッドを引き直し |
| グリッドに再配置 | 外部コード進行を現在のグリッドへ再配置（編集がある場合は確認） |
| 選択コードの開始/終了を現在位置に | `I` / `O` |
| ループ | `B` / `E` で開始/終了、`L` でON/OFF |
| 保存 | `S`（自動保存もあり） |
| 再生/停止・シーク | `Space` / `←` `→` |

## コード推定パイプライン (3層)

### Layer 1: 外部コード情報

```
YouTube URL → 動画情報取得 (oEmbed/watchページ)
  → 曲名・アーティスト名推定 (不要語除去 + J-POP表記パターン)
  → 表記揺れを考慮した検索クエリ生成
      「{曲名} {アーティスト} コード / ギターコード / ピアノコード / 弾き語り / chords」
      + 全角半角・スペース有無・記号除去・括弧内別表記のバリアント
  → 検索の多層実行
      - コードサイトへの直接検索 (U-FRET / ChordWiki)
      - Web検索エンジン (DuckDuckGo / Bing) を複数クエリで
  → 検索結果を「コードページらしさ」でスコアリング
      - 加点: コードサイトドメイン (U-FRET・楽器.me・ChordWiki・easter-egg.me等) /
        コード系キーワード / 曲名一致 / アーティスト一致
      - 減点: 歌詞ページ / 販売ページ / ニュース / 動画・配信ページ (理由つきで除外記録)
  → 上位ページを取得し4段構えで抽出
      (①chordマークアップ ②[C]表記 ③コード密集行 ④JS埋め込みデータ走査)
      ♭・全角・N.C.・m7-5等の表記を正規化、カポ・キー表記も検出
  → 複数ソース照合
      - カポ表記は実音キーへ事前補正
      - キー違いは12通りの移調を試して最も一致する移調量で照合
      - 一致するコードは信頼度アップ、複数ソースあるのに不一致なら「要確認」
```

見つからなければ**進行を生成せず**、案内UIを出します。曲名・アーティスト名を手動修正して再検索もできます。外部サイトの内容は初期コード候補を作るための参考情報としてのみ使い、保存するのはコード名の列と出典URLだけです。

**🔍 検索デバッグ表示**: 推定曲名・生成クエリ・検索プロバイダーごとのヒット数/エラー・候補URLの採用/除外理由・抽出コード数・キー補正・所要時間を画面上で確認できます（開発用の折りたたみパネル）。

### Layer 2: 音源解析 (ブラウザ内で完結)

アップロードされた音源をWeb Audioでデコードし、自前DSP (STFT 4096/hop 512 @11kHz) で解析します。実際のミックス (ドラム・ボーカル・残響を含む) での精度を上げるため、以下を行っています。

- **周波数分解能の確保**: FFT窓を4096点に拡大し、低音域でも隣接する半音を区別できるようにした上で、ビン周波数が2つのピッチクラスの中間にある場合は両方に按分してchromaへ加算（低音域ほどビン幅が半音間隔に対して相対的に広いため、丸め誤差を減らす）
- **簡易HPSS (harmonic-percussive分離)**: 時間方向メディアン（持続音を残す）と周波数方向メディアン（広帯域な打楽器成分を残す）の二乗比からソフトマスクを作り、ドラム等の瞬間的な非調性成分を抑制。振幅は対数圧縮 (log1p) してボーカルや打楽器の突出したピークの影響も弱める
- **BPM**: 対数圧縮したスペクトラルフラックスの自己相関 + J-POPテンポ帯の重み付け
- **拍位置**: 位相コム探索（拍時刻はフレーム中心=アタック位置基準）
- **小節頭**: 拍上のオンセット強度 + 和声変化 (chroma距離) のスコアリング、曲頭への外挿つき
- **コード候補**: **小節を基本単位**としてメジャー/マイナー/7th/6th/sus4/sus2/dim/augの11種×12ルートのテンプレートと照合する（多くのJ-POPは1小節1コードで進行するため）。拍単位まで細かく判定すると、ノイズによる誤検出で細切れになり逆に精度が落ちることが分かったため、小節の前半/後半が明確に異なるコードだと判断できた場合に限り、その小節だけ半小節に分割する（分割は「小節→半小節」の1段階のみ）。集約は合計ではなく中央値を使い突発ノイズの影響を抑制。ベースchroma優勢音でオンコード判定
- グリッド信頼度 (high / medium / low) をUIに表示
- 1小節1コードの安定した進行・半小節ごとのコード変化・打楽器ノイズ・干渉するメロディを模した合成音声でのテストにより、過剰な細分化が起きないこと、およびそれぞれ高い精度を確認済み（`npm test`）

### Layer 3: 統合・補正

- 外部コード列を小節単位でグリッドに配置（コード数:小節数の比率で1小節2コード〜1コード4小節を自動判断）
- 音源解析のコード候補と照合し、**一致で信頼度アップ / 矛盾で信頼度ダウン+「⚠ 要確認」**
- 音源未解析のときは「仮定グリッド」(コード数から推定したBPMをJ-POP帯に折り込み) を使い、タイミング信頼度 low と明示

### 信頼度の考え方

内部的に全コードが `high / medium / low / unknown` と根拠 (evidence) を持ちます。

- 外部ソース複数一致 + 実測グリッド: **high**
- 外部ソース1件 + 実測グリッド: **medium**
- 音源解析のみ: **low〜medium**
- 手動編集済み: **high** (最優先)
- 根拠なし: **表示しない**

低信頼のコードはタイムライン上で半透明+イタリック、要確認は破線枠+⚠で、完成品のように見せません。

## MVPでできること

- YouTubeリンク入力・埋め込み再生・タイトル/チャンネル/動画長の取得
- 曲名・アーティスト名推定と手動修正・再検索
- 外部コード譜の検索・抽出・複数ソース照合
- 音源ファイルアップロードによるBPM・拍・小節グリッド・コード候補・ベース音の推定 (解析進捗表示つき)
- 拍・小節グリッドのタイムライン表示とスナップ編集 (OFF/拍/小節)
- 解析結果サマリー (外部ソース数 / 推定BPM / タイミング信頼度 / 要確認数)
- YouTube同期のコード表示・ピアノ鍵盤表示 (左手/右手、オンコードは slash後の音を左手ベースに)
- コード音再生 (両手/左手/右手、音量・長さ調整)、原曲＋コード同期再生、コードのみ再生
- 任意区間ループ (全モード)
- コード・タイミング編集、全体シフト、キーボードショートカット
- 曲ごとのプロジェクトをlocalStorageに自動保存 (旧形式からの自動マイグレーションつき)
- **ライト/ダークテーマ切り替え** (OS設定を初期値に、選択を保存)

## 現在の制限

- **テンポ一定を仮定**: 拍グリッドは一定BPMで引くため、テンポが揺れる曲・転調やリタルダンドには弱い（全体シフト+タップ修正で補正する前提）
- **音源解析は簡易DSP**: 7th/6th/sus/dim/augまで対応していますが、9th以上のテンションや複雑な転回形は取れません。ボーカルが非常に大きく持続的に非構成音を歌い続ける等、極端な条件では精度が下がります
- **半小節より細かいコード変化は検出しません**: 小節を基本単位とし、明確な場合のみ半小節に分割する設計のため、1小節に3つ以上コードが入るような曲や、分割の判断が微妙なケース（近親調の和音同士など）は1つのコードにまとめられることがあります。その場合はタイムライン編集で手動修正してください
- **YouTube音声の直接解析はしない**: 規約上ストリームを取得しないため、音源解析はユーザーの音源ファイルが必要です
- **外部サイト構造変化に弱い**: スクレイピングベースのため取得できないことがあります（その場合も定番進行は出さず、案内UIに切り替わります）
- **コード音は簡易シンセ**、保存はブラウザローカルのみ

## 技術構成

- **Next.js 15 (App Router) + TypeScript + React 19**、外部UI/DSPライブラリなし
- 分離モジュール: 外部コード推定 `src/lib/chordSources/` / 音源解析 `src/lib/audioAnalysis/` (純TSでNodeテスト可) / 統合 `src/lib/integrate.ts` / コード理論 `src/lib/chords.ts` / タイムライン・スナップ `src/lib/timeline.ts` / 音声 `src/lib/audio.ts` / 同期 `src/hooks/useSyncEngine.ts` / 保存 `src/lib/storage.ts`
- 同期は「モードごとに1つのクロック」に統一し、rAFでコード境界・ループ・シークを検知。シーク/ループ/停止時は必ず発音停止→再トリガー
- テーマはCSS変数 + `data-theme` 属性。初回描画前にインラインスクリプトで適用しちらつきを防止
- `npm test` で合成音声によるBPM/拍/コード検出の検証を含む61ケースを実行
